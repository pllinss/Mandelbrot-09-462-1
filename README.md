
# Лабораторная работа №3
### Командный проект Фрактал «Множество Мандельброта»
Создать графическое приложение с набором классов для вывода в окне
заданной пользователем области точек комплексной плоскости. Цвета точек
должны зависеть от их принадлежности множеству Мандельброта.

## **Требования к проекту. (уже реализовано)**

**0.1.** Подключить к проекту модуль с классом-преобразователем координат в jarформате, созданного в предыдущей лабораторной работе.

**0.2.** Осуществить отрисовку фрактала в цвете. Цвет точки выбирать согласно формулам, обеспечивающим полноцветность изображения. Формулы должны быть различными для каждой цветовой составляющей (RGB) и обеспечивать нелинейную зависимость от доли успешно пройденных итераций по проверке принадлежности точки множеству.

**0.3.** Обеспечить возможность масштабирования фрагмента изображения множества Мандельброта, выделенного пользователем.

## Гайд по работе с Git в IntelliJ IDEA
1. Сделать **Fork** репозитория
- Открой браузер и перейди в репозиторий
https://github.com/yotisha842/Mandelbrot-09-462-1
- Нажми кнопку **Fork** (в правом верхнем углу)
- Выбери свой аккаунт → нажми **Create fork**

2. Клонировать форк на компьютер
- Открой IntelliJ IDEA
- Нажми троеточие в левом верхнем углу File -> Close Project (если в данный момент открыт другой проект)
- Нажми кнопку **Clone Repository** (справа сверху)
- Зайди в свой GitHub в репозиторй с проектом который был форкнут
- Нажми на зеленую кнопку **<> Code** и выбери HTTPS
- Скопируй ссылку и вставь в Intelji Idea в поле URL 
- Если нужно измени путь сохранения (нужно чтобы в папке в которую вы хотите сохранить проект ничего не было)
- Нажми Clone

3. Работа с кодом
- Чтобы сохранить изменения -   Нажми `Ctrl + K` (`Git` → Commit...) → откроется окно коммита
- Выбери файлы для коммита (поставь галочки) либо ничего не меняй если они уже проставлены
- Напиши сообщение для коммита которое кратко описывает что было рализовано
- Нажми **Commit and Push**
- Снизу справа может вылезти предупреждение нужно выбрать пуш в любом случае

4. Синхронизация с первоначальным репозиторием (Sync Fork)
**Это делается КАЖДЫЙ РАЗ ПЕРЕД НАЧАЛОМ РАБОТЫ**
Могли быть приняты другие Pull Request'ы, и твой форк устарел. Нужно обновить его.
- Зайди на страницу своего форка на GitHub
- Нажми кнопку **Sync fork**
- Если есть новые изменения, нажми **Update branch**
- Потом перейти в IntelJI Idea и нажми троеточие Git -> pull

5. Создание Pull Request
- Зайди на страницу своего форка на GitHub
- GitHub сам предложит создать PR, если ты недавно пушил
- Нажми **Compare & pull request**


Создать ветку для задачи (не обязательно)
**Перед началом написания кода создайте свою светку**
- В левом верхнем углу нажми на текущую ветку (сейчас `main`)
- Выбери **New Branch**
- Введи название ветки по шаблону: feature/название-задачи
- Нажми `Create`
- Сверху должно отобразиться вместо main название новой ветки
